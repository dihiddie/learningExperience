О системе 

  Назначение системы – моделирование и решение нестандартных задач анализа, управления и обработки информации на основе применения современных информационно-аналитических технологий поддержки принятия решений.

 

Глоссарий

Модель

Запрограммированный в системе алгоритм, на основе которого производится обработка входных данных и вывод результата моделирования для решения необходимой задачи.

Статусы модели:

  • Черновик – модель в стадии настройки/тестирования алгоритма
  • Активно – настроенная модель, готовая к использованию для получения результата моделирования
Архив – неактивный статус модели, модель невозможно использовать для запуска процесса моделирования
Версия модели

Состояние модели с определенным набором узлов и параметров.

Статусы версии модели по готовности к действию:

  • Черновик – версия модели в стадии настройки/тестирования алгоритма
  • Готов к запуску – версия модели с настроенным набором узлов и параметров, готовая к запуску процесса моделирования
Архив – версия модели в неактивном состоянии, выполнение действий с моделью в данной версии не представляется возможным
Эпизод

Составная часть процесса моделирования, характеризующаяся набором настроенных параметров определённой модели, позволяющая производить вычисление на основе запрограммированных в системе алгоритмов.

Статусы эпизода:

  • Черновик – эпизод модели в стадии настройки/тестирования алгоритма
  • Актуальный – настроенный эпизод модели, готовый к использованию для получения результата моделирования
  • Архив – неактивный статус эпизода модели, эпизод невозможно использовать для запуска процесса моделирования
Узел

Составная часть модели с набором параметров и данных, необходимых для преобразования входного пакета данных и/или вывода промежуточного/итогового выходного результата моделирования согласно заложенному алгоритму.

Типы узлов:

  • Вычислительный – предназначен для обработки входящего пакета данных согласно алгоритму
  • Корректировочный – предназначен для просмотра результата моделирования (исходящего пакета) в узле

Список статусов (анкор)
Версия узла Вариация структуры узла. Структура версии узла состоит из фиксированного набора скриптов обработки и параметров.
Экземпляр узла

Событие, хранящее историю об одном запуске узла модели.

Инстансы узла:

  • Установка активности узла – запуск вычислений в узле
  • Приостановка узла – временная остановка вычислений в узле с последующей возможностью продолжения
Прерывание работы узла (при открытом эпизоде) – полная остановка вычислений в узле без возможности продолжения в рамках данного экземпляра узла
Связь Маршрут расчетов от одного узла к другому. В интерфейсе системы представляет собой стрелку, соединяющую два и более узлов.
Пакет данных Набор файлов, сформированный в ходе выполнения скрипта, либо загруженный пользователем при работе с пользовательским интерфейсом системы
Исполнитель Аппаратные средства для выполнения сценария скрипта.
Спецификация  Соединительный элемент двух и более узлов , несущий информацию о составе, структуре, месте хранилища и маршрутах пакета данных
Параметры

Задаваемые характеристики узла.

  • Глобальные – параметры, общие для всех узлов модели
  • Локальные – параметры, задаваемые для конкретного узла модели
Скрипт

Последовательность действий, описанных с помощью скриптового языка программирования для автоматического выполнения вычислений алгоритма при запуске узлов модели при наличии входного пакета данных.
Версия скрипта Вариация структуры и состава скрипта. Представляет собой порядковый номер вариации структуры скрипта в рамках одной модели.
Готовое решение Комплекс настроенных узлов, связей, глобальных/локальных параметров, скриптов, составляющих модель для решения задачи.
Панель АРМ Рабочее пространство системы, в котором производится моделирование с возможностью настройки инструментов пользовательского интерфейса (виджетов) нужным образом.
